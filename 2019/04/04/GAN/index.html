GAN 深度学习 图像处理

深度学习进阶

"GAN环境搭建及图像处理应用"

Posted by Xudong on April 4, 2019

前言

本文我们就以linux下的深度学习算法入手，从环境配置开始，感受对抗生成网络GAN的性能效果。CSGM和DCGAN是两种典型的基于GAN设计的作用于图像处理的深度学习算法，按照本文的操作流程，读者就可以完成这两个网络的环境搭建和运行。

硬件和系统需求

  1. 英伟达显卡，显卡的计算能力 >= 3.0，性能越高越好，查询显卡的计算能力的官方链接：https://developer.nvidia.com/cuda-gpus 。经过本人手动验证GTX 780ti以上的显卡可以确保满足需求。
  2. 系统为Ubuntu，它是一种基于linux开发的免费开源操作系统。大部分的深度学习算法都以这个系统环境下开发为主，之后的大部分算法介绍和应用也将以这个系统环境下进行详细展开。当前业界最为广泛使用的是ubuntu 16.04 64位系统版本，本文也将在这个版本下进行环境搭建。不熟悉ubuntu系统的同学可以先从安装系统学起，留出1~2天的入门学习时间就可以。

    环境搭建

    0. 禁用集成显卡

    在开始安装之前需要提醒以下电脑是笔记本的同学，极有可能电脑是双显卡，因此需要将飞英伟达的那块显卡禁用掉才能进行以下操作。当然如果只安装了一块显卡的台式机，可以直接跳过这一步，正式开始安装环境。 Nouveau是Ubuntu自带的第三方开源N卡驱动，安装NVIDIA官方驱动之前需要禁用Nouveau.

（1）删除所有已经安装的nvidia驱动（如果系统是新装的跳过这个步骤）

1

$ sudo apt-get remove nvidia* && sudo apt autoremove

（2）安装一些依赖库

$ sudo apt-get install dkms build-essential linux-headers-generic

（3）禁用Nouveau 终端输入sudo gedit /etc/modprobe.d/blacklist.conf在弹出的文本文件中增加下面几句语句

1
2
3
4
5

blacklist nouveau
blacklist lbm-nouveau
options nouveau modeset=0
alias nouveau off
alias lbm-nouveau off

然后保存并退出。继续输入

1
2
3

$ echo options nouveau modeset=0
$ sudo tee -a /etc/modprobe.d/nouveau-kms.conf
$ sudo update-initramfs -u

重启$ reboot

（4）通过这个命令lsmod | grep nouveau来确认系统是否已经禁用Nouveau，如果没有任何反馈结果，说明已经禁用成功。

1. 安装GPU版本的tensorflow

ubuntu 16.04 系统自带python 2.7，我们可以通过在终端输入python来验证，并且查看版本号，退出python则输入exit()。

（1）安装pip

接下来正式安装大名鼎鼎的tensorflow，选用pip安装的方式最为便捷。pip是安装python拓展软件包的一个插件，安装命令如下：

1
2

$ sudo apt-get install python-pip python-dev #安装pip
$ sudo pip install --upgrade pip #更新pip

更换pip源为国内源，我们选用阿里云的源，在 ~/.pip/目录下生成配置文件：

1
2

$ mkdir ~/.pip/
$ gedit ~/.pip/pip.conf

然后在弹出的文档编辑器上输入

1
2
3

[global]
trusted-host=mirrors.aliyun.com
index-url=https://mirrors.aliyun.com/pypi/simple/

再保存，关闭。顺带这里提一句，系统的更新也建议切换至国内的源，如下图所示：

（2）安装GPU驱动

终端输入sudo apt-get update

如下图所示在图形界面中安装显卡驱动： System Settings →Software & Updates→Additional Drivers→Nvidia Corporation→Apply Changes→重启 验证安装，终端输入nvidia-settings，出现以下界面说明安装成功

（3）安装bazel

bazel是Google推出的用来编译tensorflow开源工具，注意这里我们不按照官方安装教程，因为过高版本的bazel会造成后期的编译问题。首先我们在终端输入以下命令：

1
2
3

$ sudo add-apt-repository ppa:webupd8team/java
$ sudo apt-get update && sudo apt-get install oracle-java8-installer
$ sudo apt-get update

验证完成安装JDK8java -version会显示所安装的java编译器版本号，接下来安装bazel，首先下载bazel安装文件，链接https://github.com/bazelbuild/bazel/releases?after=0.9.0 选择bazel-0.5.4-installer-linux-x86_64.sh下载安装脚本，然后进入下载目录cd <installpath>进行安装

1
2

$ chmod +x bazel-0.5.4-installer-linux-x86_64.sh
$ sudo ./bazel-0.5.4-installer-linux-x86_64.sh

（4）安装CUDA 8.0

这里各位也要注意了，各个软件的版本好一定要对上，并非越高级的版本越有效，有时候会出现一些难以解释的问题，都是由于版本不对导致的。这里我们选择CUDA8.0 GA2版本，下载链接：https://developer.nvidia.com/cuda-toolkit-archive 按照下图步骤选择，下载deb安装文件。 下载完毕后，用终端进入到存放安装包的目录下cd <installpath>，然后

1
2
3

$ sudo dpkg -i cuda-repo-ubuntu1604-8-0-local-ga2_8.0.61-1_amd64.deb
$ sudo apt-get update
$ sudo apt-get install cuda

（5）安装cudnn 5.1

安装包下载地址：https://developer.nvidia.com/rdp/cudnn-archive 如下图选择cuDNN v5.1 Library for Linux 下载完毕后进入下载的文件目录，解压缩，拷贝，覆盖，更新软链接

1
2
3
4
5
6
7
8

$ tar -zxvf cudnn-8.0-linux-x64-v5.1.tgz
$ sudo cp cuda/include/cudnn.h /usr/local/cuda/include
$ sudo cp cuda/lib64/libcudnn* /usr/local/cuda/lib64
$ sudo chmod a+r /usr/local/cuda/include/cudnn.h /usr/local/cuda/lib64/libcudnn*
$ cd /usr/local/cuda/lib64/
$ sudo rm -rf libcudnn.so libcudnn.so.5 #删除原有动态文件
$ sudo ln -s libcudnn.so.5.1.5 libcudnn.so.5 #生成软衔接
$ sudo ln -s libcudnn.so.5 libcudnn.so #生成软链接

（6）配置系统环境变量

在终端中输入：

1

$ sudo gedit ~/.bashrc #打开.bashrc文件

在弹出的文本编辑界面中，在文本的最后添加以下两行脚本：

1
2

export PATH=/usr/local/cuda-8.0/bin${PATH:+:${PATH}}
export LD_LIBRARY_PATH=/usr/local/cuda-8.0/lib64${LD_LIBRARY_PATH:+:${LD_LIBRARY_PATH}}

保存，并使设置立即生效，在终端中输入

1

$ source ~/.bashrc

重启电脑。

（7）安装tensorflow

在此之前安装其他依赖包

1

$ sudo apt-get install python-numpy python-dev python-wheel

下载tensorflow-1.5.0的源码，注意千万不要下载官网的代码，因为官网已经更新至tensorflow-2.0，对应的软件环境配置也会翻天覆地的变化。因此我们选用更老的版本，如tensorflow-1.5.0，相对更加稳定，而且很多算法也是基于这样的环境编写的。

1
2

$ sudo apt-get install git
$ git clone https://github.com/theodore-zxd/tensorflow-1.5.0-rc0.git

运行配置文件

1
2

$ cd tensorflow
$ ./configure

配置完毕后开始编译，这一步耗时比较长，做好心理准备，起码一个多小时是要的：

1
2

$ bazel build --config=opt //tensorflow/tools/pip_package:build_pip_package
$ ./bazel-bin/tensorflow/tools/pip_package/build_pip_package ./tensorflow_pkg

如果上面步骤出现下面这个问题：

1

ERROR: Config value cuda is not defined in any .rc file

这个就是你之前安装的bazel版本过高导致的，所以一定要选择对应的bazel-0.5.4的版本才行。解决方法就是重新安装bazel即可，安装脚本会自动覆盖之前的版本，不需要卸载。

成功的话会在，在tensorflow_pkg下生成了tensorflow-1.5.0rc0-cp27-cp27mu-linux_x86_64.whl然后进入到这个文件夹下，运行以下命令，用pip完成安装：

1

$ sudo pip install tensorflow-1.5.0rc0-cp27-cp27mu-linux_x86_64.whl

3. 搭建CSGM和DCGAN

（1）安装pypng和pywt

1
2

$ sudo pip install pypng #这是python处理png图像的拓展包
$ sudo pip install PyWavelets #这是python进行小波变换的拓展包

（2）下载源码

1
2
3

$ git clone https://github.com/AshishBora/csgm.git
$ cd csgm
$ git submodule update --init --recursive

（3）下载数据集

1

$ ./setup/download_data.sh

（4）下载已经训练好的模型

1

$ ./setup/download_models.sh

（5）运行demo

1

$ ./quick_scripts/celebA_superres.sh "./images/182659.jpg"

这段demo实现的功能是图像超分辨率，运行结果如下图所示： 三张图的上图是原高分辨率图像，中间则是通过对第一张图做降低分辨率获得的低分辨率图像，最后一张是算法通过深度学习网络计算获得的高分辨率图像。它实现了将模糊的人脸变清晰的功能，但是我们发现变出的脸更原来的图像大相径庭。这也是这套算法，或者说对抗生成网络GAN的通病。

我们再尝试另一段demo:

1

$ ./quick_scripts/celebA_inpaint.sh "./images/182659.jpg"

这段demo完成的是inpainting的工作，也就是将原始图像挖去一块，用GAN来进行填充，这样算法就可以在某些特定的领域完成神奇的功能：）。但是我们在下图结果中可以发现，算法的性能有待改进。 但是对于数字来说，这样的 性能已经完全足够了，终端中输入以下命令：

1

$ ./quick_scripts/mnist_inpaint.sh

我们遮住数字的一部分，完全不影响算法识别出数字的原来的样子，这样的性能在某些工业领域完全足够能够胜任。

（6）训练

CSGM的训练代码是调用DCGAN，因此我们需要转向研究DCGAN模型。

（1）下载DCGAN源代码

DCGAN有很多版本的代码，这里我们选择tensorflow版本的，参考链接为https://github.com/carpedm20/DCGAN-tensorflow 输入以下命令下载源码：

1

$ git clone https://github.com/carpedm20/DCGAN-tensorflow.git

（2）添加mnist数据集

在DCGAN官方链接中提示我们需要在线下载数据库，但是在国内的同学使用脚本下载数据库会出现网络问题，因为google被墙了。所幸我们在上个部分CSGM中已经下载过这个数据集，只要把他们挪过来就可以使用了（注意mnist挪过来的时候里面的压缩包都要解压，然后每个文件的名字都需要将中间的.改成-）。

（3）训练和测试

在DCGAN根目录下的终端中输入

1

$ python main.py --dataset mnist --input_height=28 --output_height=28 --train

训练需要一些时间，并且显卡的动静很大。完成以后，输入以下指令完成测试：

1

$ python main.py --dataset mnist --input_height=28 --output_height=28

测试结果的图像输出在/samples目录下，如下图所示。而训练的网络参数保存在、checkpoint目录下，后面大家可以自己试下用DCGAN的训练结果去替换之前CSGM的网络参数（因为CSGM就是基于DCGAN设计的），再运行下CSGM，看看自己训练的模型输出如何。

总结

由于篇幅问题，linux下深度学习算法初步入门就讲到这里，后面读者可以在CSGM和DCGAN的基础上，深入研究这两个算法的机理。

此外，完成本文环境配置过程的同学一定觉得配置算法环境很麻烦，而且如果我们想切换另一套代码，换个环境，那岂不是又要重装系统，再来一遍？非也，后面的文章我们就会一起研究解决这个问题的好方法——docker，敬请关注。
